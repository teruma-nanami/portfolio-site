# ポートフォリオサイト要件定義

## 目的・背景

- 就職・転職活動のため、自身のスキルや実績を採用担当者・企業にアピールするポートフォリオサイトを作成する。
- あわせて、本サイト自体の開発をIssue駆動・AI駆動開発で進め、その取り組み自体も成果としてアピールする。

## 利用者

- 主に採用担当者・企業の採用関係者を想定する。

## 必要な挙動

- プロフィール・自己紹介ページで、自己紹介文・経歴・スキルなどを閲覧できる。
- 制作実績一覧ページで、実績のタイトル・概要を一覧表示し、各項目から既存の外部サイト（詳細URL）へ遷移できる。
- 開発プロセス紹介ページで、本サイトのGitHubリポジトリへのリンク、Issue駆動・AI駆動開発の取り組み説明、使用技術スタックの紹介を閲覧できる。
- お問い合わせフォームから氏名・メールアドレス・問い合わせ内容を入力して送信すると、運営者宛にメール通知が送信される。

## 受け入れ条件

- [ ] プロフィール・自己紹介ページで、自己紹介文・経歴・スキルを閲覧できる。
- [ ] 制作実績一覧ページで、実績のタイトル・概要が一覧表示され、各項目から外部の詳細URLへ遷移できる。
- [ ] 開発プロセス紹介ページで、GitHubリポジトリへのリンク・Issue駆動/AI駆動開発の説明文・使用技術スタックの紹介を閲覧できる。
- [ ] お問い合わせフォームに氏名・メールアドレス・問い合わせ内容を入力して送信すると、運営者へメール通知が届く。
- [ ] お問い合わせフォームの入力に不備がある場合、バリデーションエラーが表示され、入力内容が保持される。
- [ ] 各ページがスマートフォン・PCの両方で適切に表示される。

## 対象外

- 管理画面・ログイン機能
- ブログ・記事機能
- 多言語対応
- Reactなど別フロントエンドフレームワークの導入(Laravel Blade + Tailwind CSSのみで実装する)
- 制作実績データそのものの管理(実績詳細は既存の外部サイトで管理する)
- お問い合わせ内容のDB保存

## 制約

- アプリケーション構成はLaravel単体(Blade + Tailwind CSS)とする。
- データベースはSQLiteを使用する。
- 開発環境はLaravel Sail(Docker)を使用する。
- お問い合わせの通知メール送信にはMailableを使用する。
